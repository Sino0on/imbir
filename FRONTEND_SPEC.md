## Авторизация

Все новые эндпоинты требуют JWT токен.

**HTTP запросы:**
```
Authorization: Bearer <ACCESS_TOKEN>
```

**WebSocket** — токен передаётся в query-параметре (не в заголовке):
```
wss://домен/ws/chat/{room_id}/?token=ACCESS_TOKEN
```

## Чат между пользователями

### База
- HTTP: `https://домен/api/chat/`
- WebSocket: `wss://домен/ws/chat/{room_id}/?token=ACCESS_TOKEN`

---

### GET `/api/chat/rooms/`
Список всех комнат текущего пользователя.

**Ответ 200:**
```json
[
  {
    "id": 3,
    "participants": [
      { "id": 1, "full_name": "Даниял Бек" },
      { "id": 5, "full_name": "Айгуль Мамытова" }
    ],
    "last_message": {
      "content": "Привет!",
      "created_at": "2026-06-28T10:01:00Z"
    },
    "created_at": "2026-06-28T10:00:00Z"
  }
]
```

---

### POST `/api/chat/rooms/`
Создать или получить существующую комнату с другим пользователем.
Если комната уже есть — вернёт её (не создаст дубликат).

**Запрос:**
```json
{ "user_id": 5 }
```

**Ответ 201:**
```json
{
  "id": 3,
  "participants": [
    { "id": 1, "full_name": "Даниял Бек" },
    { "id": 5, "full_name": "Айгуль Мамытова" }
  ],
  "last_message": null,
  "created_at": "2026-06-28T10:00:00Z"
}
```

---

### GET `/api/chat/rooms/{id}/messages/`
История сообщений комнаты. Входящие непрочитанные автоматически помечаются прочитанными.

**Ответ 200:**
```json
[
  {
    "id": 1,
    "sender": { "id": 1, "full_name": "Даниял Бек" },
    "content": "Привет!",
    "created_at": "2026-06-28T10:01:00Z",
    "is_read": true
  }
]
```

---

### WebSocket

**Флоу подключения:**
1. POST `/api/chat/rooms/` с `user_id` → получаем `room.id`
2. Загружаем историю: GET `/api/chat/rooms/{id}/messages/`
3. Открываем WS: `wss://домен/ws/chat/{room_id}/?token=ACCESS_TOKEN`
4. Слушаем входящие события, отправляем сообщения

**Коды закрытия соединения:**
| Код | Причина |
|-----|---------|
| 4001 | Невалидный или отсутствующий токен |
| 4003 | Пользователь не является участником комнаты |

**Отправка (клиент → сервер):**
```json
{ "content": "Текст сообщения" }
```

**Входящее событие (сервер → все участники комнаты):**
```json
{
  "id": 42,
  "sender": { "id": 3, "full_name": "Айгуль Мамытова" },
  "content": "Текст сообщения",
  "created_at": "2026-06-28T10:05:33.123456"
}
```

> ⚠️ Сообщение отправителя тоже приходит обратно через WS (echo).
> Нужно проверять `sender.id === currentUser.id` чтобы не дублировать в UI.

## ИИ-чат с врачом (комната 0)

Отдельный чат без WebSocket — работает по принципу запрос/ответ.
Доступен **только авторизованным пользователям**.
Каждый пользователь видит только свою историю.

---

### GET `/api/chat/ai/`
История чата с ИИ текущего пользователя.

**Ответ 200:**
```json
[
  { "id": 1, "role": "user",      "content": "У меня болит голова", "created_at": "..." },
  { "id": 2, "role": "assistant", "content": "Как давно болит?..",   "created_at": "..." }
]
```

---

### POST `/api/chat/ai/send/`
Отправить сообщение ИИ-врачу. Возвращает только ответ ассистента — сообщение пользователя сохраняется автоматически.

> ⏳ Запрос может занять **3–10 секунд**. Обязательно показывать лоадер.

**Запрос:**
```json
{ "message": "У меня болит голова" }
```

**Ответ 201:**
```json
{ "id": 2, "role": "assistant", "content": "Как давно болит?...", "created_at": "..." }
```

---

### DELETE `/api/chat/ai/`
Очистить историю чата с ИИ. Ответ: `204 No Content`.

## Онлайн-запись с Google Meet

При создании записи с `is_online: true` сервер автоматически генерирует Google Meet ссылку.

> ⚠️ Онлайн-запись доступна **только авторизованным пользователям**.
> Гость с `is_online: true` получит ошибку `400`.

---

### POST `/api/appointments/`

**Запрос:**
```json
{
  "doctor_id": 12,
  "date": "2026-07-10",
  "time": "14:00:00",
  "is_online": true
}
```

**Ответ 201 — поле `google_meet_link` заполнено:**
```json
{
  "id": 55,
  "date": "2026-07-10",
  "time": "14:00:00",
  "is_online": true,
  "google_meet_link": "https://meet.google.com/abc-defg-hij",
  "status": "pending",
  "doctor": { "id": 12, "full_name": "Алмаз Токтосунов" },
  "..."
}
```

### Новые поля в ответе `/api/appointments/`

| Поле | Тип | Описание |
|------|-----|----------|
| `is_online` | boolean | Онлайн или офлайн запись |
| `google_meet_link` | string \| null | Ссылка на Meet. Заполнена только если `is_online: true` |

### UX

- Чекбокс «Онлайн-приём» показывать только авторизованным пользователям
- После создания записи с `is_online: true` — показать кнопку **«Открыть Google Meet»** со ссылкой из `google_meet_link`
- В карточке записи всегда показывать кнопку если `is_online === true && google_meet_link !== null`
